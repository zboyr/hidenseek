\begin{abstract}
Different language models leave distinct stylistic traces---``fingerprints''---when paraphrasing text.
We investigate whether these fingerprints are detectable by designing an adversarial evaluation framework in which an \emph{Auditor} generates challenging source texts, four diverse paraphrasers rewrite them, and a \emph{Detective} attempts to identify which two outputs originate from the same paraphraser.
Across 10 independent runs totaling 150 scored trials, the Detective achieves a mean accuracy of 60.7\% (95\% CI: [52.1\%, 69.2\%]), significantly above the 10\% random baseline ($t(9)=13.41$, $p < 0.001$).
Detection rates vary markedly across paraphrasers: GPT-5-Nano is identified with 100\% accuracy, while Llama-3.1-8B proves hardest to detect at 25\%.
These results confirm that paraphraser fingerprinting is feasible and that different model architectures produce distinguishable rewriting patterns.
\end{abstract}

\section{Problem Description}
% ============================================================

Large language models (LLMs) are increasingly used to paraphrase, rewrite, and polish text.
A natural question arises: \emph{do different models leave distinct traces when they paraphrase?}
If so, it may be possible to identify \emph{which} model produced a given rewrite---analogous to identifying the ``wood grain'' of an AI model.

This question has practical implications for AI-generated text detection, academic integrity, intellectual property, and understanding model behavior.
Prior work on AI text detection has focused on distinguishing human-written from machine-generated text~\cite{mitchell2023detectgpt}, but the finer-grained problem of \emph{paraphraser attribution}---identifying which specific model performed a rewrite---remains underexplored.

We formalize this as a \textbf{paraphraser fingerprinting} task:
given a set of paraphrased outputs from multiple models (where one model contributes two outputs and the rest contribute one each), can a detector identify the two outputs that share the same origin?
This framing isolates stylistic fingerprints from content, since all outputs derive from the same source text.

Our research question is:
\begin{quote}
\textbf{RQ: Paraphraser Fingerprints.} Do different paraphrasing models leave distinct, detectable traces in their outputs? Can we identify the paraphraser from the text alone?
\end{quote}

% ============================================================
\section{Dataset}
% ============================================================

We do not use a pre-existing dataset.
Instead, we generate all experimental data dynamically through our adversarial evaluation framework.
This design choice is deliberate for several reasons:

\begin{enumerate}
    \item \textbf{No suitable benchmark exists.} Existing paraphrase datasets (e.g., MRPC, QQP) contain human-written paraphrase pairs, not outputs from multiple competing paraphrasers applied to the same source.
    \item \textbf{Source text diversity matters.} The detectability of fingerprints depends heavily on the source text. Our framework generates source texts specifically designed to elicit distinguishable behaviors from different paraphrasers.
    \item \textbf{Adversarial refinement.} Through an iterative Auditor--Detective feedback loop, source texts are progressively refined to maximize fingerprint visibility, creating a more challenging and informative evaluation than static benchmarks would allow.
\end{enumerate}

Each experimental run produces 20 trials.
Only the initial trial uses a human-crafted source text containing deliberate linguistic traps (non-standard grammar, intentional repetition, temporal disorder, counting exercises).
All subsequent trials use source texts generated by the Auditor LLM using Automatic Prompt Engineering (APE)-style iterative refinement based on past Detective feedback.

\section{Experimentation and Results}
% ============================================================

\subsection{Experimental Setup}

\paragraph{Framework Architecture.}
Our system consists of three roles:

\begin{itemize}
    \item \textbf{Auditor} (Claude Sonnet 4.6): Generates source texts designed to elicit distinguishable paraphrasing behaviors. Receives feedback from the Detective's performance to iteratively refine its strategy.
    \item \textbf{Detective} (Claude Sonnet 4.6): Analyzes a set of 5 paraphrased outputs (from 4 paraphrasers, one contributing two outputs) and identifies which two share the same origin. Operates statelessly in most configurations.
    \item \textbf{Paraphrasers} (4 models):
    \begin{itemize}
        \item Llama-3.1-8B-Instruct (via OpenRouter)
        \item Gemini-2.5-Flash (via OpenRouter)
        \item GPT-5-Nano (via OpenRouter)
        \item DIPPER (T5-based, via HuggingFace Endpoint, $\text{lex}=20$, $\text{order}=40$)
    \end{itemize}
\end{itemize}

\paragraph{Trial Protocol.}
Each trial proceeds as follows:
\begin{enumerate}
    \item The Auditor generates (or selects) a source text.
    \item One paraphraser is randomly chosen to produce \emph{two} outputs; the remaining three each produce one output, yielding 5 total outputs.
    \item The outputs are shuffled randomly.
    \item The Detective examines all 5 outputs and predicts which pair originated from the same paraphraser.
    \item A results block (correct/incorrect, predicted vs.\ actual indices) is fed back to the Auditor.
\end{enumerate}

\paragraph{Configurations.}
We ran 10 independent experiments, each consisting of 20 trials with 5 warmup trials (not scored).
Two configurations were tested: (1) a separate Detective with no memory across trials ($n=2$ runs), and (2) the Auditor serving as Detective with access to past outputs and results ($n=8$ runs).

\paragraph{Baseline.}
Random guessing yields $\frac{1}{\binom{5}{2}} = \frac{1}{10} = 10\%$ accuracy, since the Detective must select 2 of 5 outputs.

Across 10 independent runs (150 total scored trials), the system achieves a mean detection accuracy of \textbf{60.7\%} (SD = 0.12), with a 95\% confidence interval of [52.1\%, 69.2\%].
A one-sample $t$-test against the random baseline confirms statistical significance: $t(9) = 13.41$, $p < 0.001$ (Figure~\ref{fig:accuracy_dist}).

Individual run accuracies range from 46.7\% to 86.7\%, reflecting the inherent stochasticity of random paraphraser duplication assignment and LLM output variability.

Detection rates vary dramatically across paraphrasers (Figure~\ref{fig:detection_rate}):

\begin{itemize}
    \item \textbf{GPT-5-Nano}: 100\% (36/36)---always detected when duplicated.
    \item \textbf{Gemini-2.5-Flash}: 64.3\% (18/28).

    \item \textbf{Llama-3.1-8B}: 25.0\% (6/24)---hardest to detect.
\end{itemize}

GPT-5-Nano's perfect detection rate suggests it produces highly consistent, distinctive paraphrases---its ``fingerprint'' is strong and stable.
Llama-3.1-8B's low detection rate indicates high output variability, making its two outputs less recognizably similar to each other.

The lack of improvement suggests that within-trial stylistic comparison is more informative than cross-trial memory, or that the current memory mechanism does not effectively leverage historical information.

\section{Plan for Next Steps}
% ============================================================

Based on our findings, we propose the following directions:

\begin{enumerate}
    \item \textbf{Upgrade Detective model.} Replace Claude Sonnet 4.6 with a more capable model (e.g., Claude Opus, GPT-5) to test whether stronger reasoning improves fingerprint detection.
    \item \textbf{Prompt optimization.} Systematically optimize the Detective's prompt using techniques such as chain-of-thought prompting, few-shot examples of successful identifications, or structured comparison templates.
    \item \textbf{Expand paraphraser pool.} Add more paraphrasers (e.g., Mistral, Command-R, Qwen) to test whether fingerprinting scales to larger model sets and whether models from the same family (e.g., Llama-3.1-8B vs.\ Llama-3.1-70B) are distinguishable.
    \item \textbf{Feature-based detection.} Complement the LLM-based Detective with traditional NLP features (n-gram distributions, syntactic parse patterns, vocabulary richness metrics) to build an interpretable fingerprint classifier.
    \item \textbf{Cross-domain evaluation.} Test whether fingerprints persist across different text domains (scientific, literary, conversational) and source text lengths.
    \item \textbf{Robustness testing.} Evaluate whether fingerprints survive post-processing steps such as grammar correction, back-translation, or additional paraphrasing.
\end{enumerate}

\section{Conclusion}
% ============================================================

We have demonstrated that different paraphrasing models leave detectable fingerprints in their outputs.
Our adversarial evaluation framework, combining an iterative Auditor with a stateless Detective, achieves 60.7\% mean accuracy across 150 scored trials---six times the random baseline of 10\% ($p < 0.001$).

Key findings include:
\begin{itemize}
    \item Fingerprint strength varies dramatically across models: GPT-5-Nano is always detectable (100\%), while Llama-3.1-8B is rarely detected (25\%).
    \item The Detective relies primarily on structural and stylistic features rather than surface-level heuristics like output length.
    \item Automated source text generation (APE-style) matches the effectiveness of human-crafted linguistic traps.
    \item Memory-augmented detection does not outperform stateless detection in our current framework.
\end{itemize}

These results suggest that the ``wood grain'' of AI models is real and measurable, opening avenues for paraphraser attribution, AI text provenance, and deeper understanding of how different architectures shape language generation.
